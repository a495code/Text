Этот метод основывается на том, что каждый браузер имеет уникальные характеристики. Технологии для создания системы управлений тикетов можно использовать различные технологии, такие как HTML, CSS, JavaScript и AJAX. HTML используется для создания структуры веб-страницы, CSS для создания стилей и внешнего вида, а JavaScript и AJAX используются для создания динамических элементов на веб-страницах. Например, для создания меню общения на веб-странице можно использовать технологию AJAX, которая позволяет загружать данные на страницу без необходимости перезагружать всю страницу. Это позволит пользователям быстро и эффективно аутентифицироваться, оставаясь на текущей странице. Фреймворки и инструментальные средства разработки Для упрощения процесса разработки и создания системы кроссдоменной идентификации можно использовать различные фреймворки и инструментальные средства. Например, можно использовать фреймворк React для создания интерфейса или фреймворк Node.js для написания серверной части приложения. Для реализации системы управления тикетов можно использовать следующие технологии: • Авторизация через социальные сети. • Общение с пользователем • Поддержка пользователя • Технологии хранения данных на стороне клиента: Сookie, - Небольшой фрагмент данных, отправленный веб-сервером и хранимый на компьютере пользователя, который содержит информацию о посещении веб-сайта. Сookies могут быть установлены с помощью JavaScript или сервера. Они используются для хранения данных о пользователе. Веб-клиент всякий раз при попытке открыть страницу соответствующего сайта пересылает этот фрагмент данных веб-серверу в составе HTTP-запроса HTML5 LocalStorage и Web Storage - это стандартные технологии хранения данных на стороне клиента. Они не требуют установки дополнительных библиотек или использования сторонних плагинов, а также доступны в большинстве современных браузеров. HTML5 LocalStorage используется для хранения данных даже после закрытия браузера, когда Web Storage хранит данные только на время открытия вкладки. Обе технологии предлагают более безопасный способ хранения данных, чем Сookie, так как не доступны для других сайтов, работая на одном домене. SessionStorage -по принципу работы эта технология похожа на Web Storage, но предназначена для хранения сеансовой информации. Данные сохраняются только на время открытия веб-страницы и удаляются, когда вкладка закрывается. Помимо этого, данные SessionStorage не доступны в новых вкладках и окнах браузера. Это код, который используется для структурирования и отображения веб-страницы и её контента. Например, контент может быть структурирован внутри множества параграфов, маркированных списков или с использованием изображений и таблиц данных. Strapi 4 -бладает рядом преимуществ, в том числе: Легкость использования: Strapi имеет простой интерфейс и удобную административную панель, что позволяет создавать и управлять API без необходимости написания кода. Гибкость: Strapi может быть настроен для работы с различными типами баз данных и может интегрироваться с различными типами фронтенда. Безопасность: Strapi включает в себя ряд функций безопасности, таких как аутентификация и авторизация, защита от атак типа DDoS и защита от перебора паролей. Docker — программное обеспечение для автоматизации развёртывания и управления приложениями в средах с поддержкой контейнеризации, контейнеризатор приложений. Позволяет «упаковать» приложение со всем его окружением[en] и зависимостями в контейнер, который может быть развёрнут на любой Linux-системе с поддержкой контрольных групп в ядре, а также предоставляет набор команд для управления этими контейнерами. Изначально использовал возможности LXC, с 2015 года начал использовать собственную библиотеку, абстрагирующую виртуализационные возможности ядра Linux — libcontainer. С появлением Open Container Initiative начался переход от монолитной к модульной архитектуре. Kubernetes — это портативная расширяемая платформа с открытым исходным кодом для управления контейнеризованными рабочими нагрузками и сервисами, которая облегчает как декларативную настройку, так и автоматизацию. У платформы есть большая, быстро растущая экосистема. Сервисы, поддержка и инструменты Kubernetes широко доступны. Название Kubernetes происходит от греческого, что означает рулевой или штурман. Google открыл исходный код Kubernetes в 2014 году. Kubernetes основывается на десятилетнем опыте работы Google с масштабными рабочими нагрузками, в сочетании с лучшими в своем классе идеями и практиками сообщества. Требования к программному продукту: 1. Интерфейс 1.1. Программный продукт должен иметь интуитивно-понятный и удобный интерфейс. 1.2. Интерфейс должен быть адаптивным для работы на различных устройствах (компьютер, планшет, смартфон). 1.3. Пользователи программного продукта должны иметь возможность создавать, просматривать, редактировать тикеты. 1.4. Необходимо предусмотреть как минимум два уровня доступа (администратор и обычный пользователь) для работы с тикетами. 1.5. Обязательно наличие подробной документации по использованию программного продукта. 2. Функциональность 2.1. Программный продукт должен позволять создавать тикеты с указанием типа запроса, описания задачи, даты выполнения, статуса выполнения и ответственного исполнителя. 2.2. Возможность фильтрации по статусу выполнения, ответственному исполнителю и типу запроса. 2.3. Пользователи программного продукта должны иметь возможность комментировать тикет. 2.4. Необходимо предусмотреть возможность отправки уведомлений о новых тикетах и изменениях в статусах тикетов на электронную почту. 2.5. Программный продукт должен иметь возможность экспорта данных в формате Excel или CSV. 3. Технические требования 3.1. Программный продукт должен быть разработан на языках программирования HTML, CSS, JavaScript и PHP. 3.2. Для хранения данных используется реляционная база данных MySQL версии 5.6 и выше. 3.3. Программный продукт должен быть разработан с использованием фреймворков Bootstrap и jQuery. 3.4. Разработка должна вестись с использованием системы контроля версий Git. 3.5. Разработка должна вестись с использованием методологии Agile. Входные данные: 1. Список требований от заказчика. 2. Исходные данные для заполнения базы данных (типы запросов, список пользователей). Выходные данные: 1. Готовый программный продукт. 2. Подробная документация по использованию программного продукта. 3. Сопроводительная документация (установка, настройка, обновление программного продукта). 1.4 Выбор разработки программного продукта Для ускорения разработки и повышения ее качества можно использовать различные сторонние сервисы и инструменты. Например: 1. Сервис контроля качества кода - Code Climate позволяет автоматически проверять код на соответствие лучшим практикам и искать потенциальные уязвимости. 2. Библиотека для тестирования - PHPUnit - это набор классов для создания и запуска тестов в PHP. 3. Инструмент автоматической сборки - Jenkins, позволяет автоматизировать сборку и тестирование кода, а также управлять развертыванием и управлением версиями. 4. Сервисы для управления базами данных - PhpMyAdmin, Adminer, Sequel Pro. Эти инструменты позволяют управлять схемами баз данных, выполнять запросы и действия с данными, создавать резервные копии и многое другое. 5. Утилита для автоматического форматирования кода - PHP-CS-Fixer. Этот инструмент помогает гарантировать соблюдение единого стиля кодирования в проекте. 6. Сервисы для совместной работы над кодом - GitHub, Bitbucket. Здесь можно хранить код и отслеживать его изменения, оставлять комментарии и дискутировать на тему будущих изменений проекта. 7. Инструмент для разработки дизайна интерфейса - Photoshop, Sketch. Продукт с отличным интерфейсом облегчит работу пользователей и повысит удобство использования продукта. Многие сервисы и инструменты можно использовать бесплатно или с маленькой оплатой, что делает решения намного более доступными для стартапов и небольших команд разработчиков. Однако не стоит забывать, что использование любого инструмента сопряжено с риском возникновения дополнительных ошибок и непредвиденных проблем, поэтому необходимо проявлять осторожность при выборе инструментов для проекта. 1.4 Разработка программного продукта от компании “DDOS-GUARD” Для разработки программного продукта для управления тикетами в рамках учебной практики мы выполнили следующие действия: 1.
